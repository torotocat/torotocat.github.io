1.npx和npm区别：npx是临时安装用完命令后卸载，npm是安装。

2.热更新：更新代码，页面自动刷新

3.错误覆盖：代码错误在浏览器上直接显示

vite内置功能：

4.webpack：把文件打包,长千个js/css合并成一个文件，减少请求次数

5.vite：启动更快，但是生态还不是特别完善，主要是webpack支持一些复杂的自定义配置，多页应用(每个页面都是独立的HTML)，每个页面都有自己独特的优化需求，比如说后台有大量js，但是用户访问频率低，自定义代码分割：有些页面不需要首屏加载，需要按需加载。还有特殊格式（md，proto）的处理，需要转换成json等。

6.ESLint + Prettier：代码规范检查，强制统一风格

7.Jset+React Testing Library:测试框架

8.环境变量：REACT_APP_开头，本质就是自定义变量的字符串

9.跨域：如果两个 URL 的协议、域名、端口有任何一个不同，它们就是 “不同源”，浏览器会阻止页面直接访问不同源的资源。这导致前端无法用本地的后端接口。解决方法：proxy配置，比如在package.json里，

10.Create React App(CRA) eject是一个命令，作用是释放和暴露webpack和babel脚本等等配置文件，此时不可逆，可以自定义配置，但是失去cra项目自动更新的东西。可以使用craco覆盖部分配置,创建 craco.config.js：

11.传统 React 应用会把所有 JS 代码打包到一个文件中（如 main.js），导致：

  • 首屏加载慢：用户必须等待整个文件下载完成才能看到页面
  • 浪费流量：即使用户只访问部分功能，也需要加载全部代码

解决方案：动态导入（Dynamic Import）

React 提供了 React.lazy 和 React.Suspense 来实现组件的懒加载：

效果

  • 按需加载：只有用户访问到 LazyComponent 时，才会加载对应的 JS 代码
  • 并行优化：Webpack 会自动把懒加载的组件拆分成独立的 chunk（如 1.chunk.js）
  • 更好的用户体验：加载过程中显示 fallback 内容（如加载中提示）

添加哈希值：每次build更新哈希值，哈希值由内容决定，内容改变网页就要重新加载
